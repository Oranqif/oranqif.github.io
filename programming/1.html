Gaming Online Store

Software: Visual Code Studio

Language: JavaScript (EJS)

Date: 15/07/2024

Introduction

Have you ever wanted an online store that focused on selling products related to gaming? You can now with this brand new website where you could purchase your physical copy of a game, a console or a figure all from the comfort of your own home. Try this website today!

About the project

When I was tasked by my lecturer to create a rudimentary online store, my thought leaned toward video games because I love playing video games. So I wanted to create an online store focused on selling gaming products.

I started by creating a Home page with a navigation bar that can navigate to six pages. The pages are laid out like this in order...

  • Home
  • Video Games
  • Consoles & Controllers
  • Figures
  • Cart
  • Contact

My implementation for the Home page



Next, to render the page, creating the route to the home page is necessary. Without it, the application would not render the page because there was no route for it thus did not know that the page existed. Also, I would have had to implement routing for every new EJS pages created. The image below was what had happened if there was no routing for the page...

This is the result of no routing for the page



Then, for the Video Game page, it needed to display an item and it has to be inside a card for readability. Not only that, the page needed to display multiple cards too without the card constricting onto themselves. To accomplish this, I created the page first that display the item by getting their ID then use CSS flexbox to create the card and a container that dynamically move the cards around depending on how many cards there are. The images below showed the Video Game page displaying items inside their respective card dynamically...

My implementation for Video Game page

See how it had displayed the items dynamically?



Next, it is time to create a page to create, edit, and delete items for the Video Game page with the RESTful api. REST is a type of software architectural style that emphasises uniform interfaces, independent employement of components and the scalability of interactions between them. In this case, it was focused on scalability as each of the items need their own page to edit. I did this by, after creating the ability to add and delete item, implementing a route that allowed one page to edit a chosen item through reading their ID. Repeat this process for the Consoles & Controllers and the Figurines pages. The image below is the page for adding, editing and deleting item...

The page that can add, edit, and delete item



Finally, the data for the items have to had come from somewhere. This is where Structured Query Language (SQL) came into place. SQL allows for persistent storage even if the website was reloaded as it was stored outside of it. The images below showed what the database looked like and how it was stored...

Conclusion

This was my first dive into utilizing a framework for JavaScript. After this project, I can see how frameworks could speed up the development of the project by making it easier to implement certain features that could be hard to do so in vanilla JavaScript. I had learnt a tons from this experience and I am willing to further expand my knowledge of Embedded JavaScript (EJS) if possible.
